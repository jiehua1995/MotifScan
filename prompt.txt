你现在在一个空目录中，从零开始创建一个高性能 Rust 命令行工具，项目名叫 MotifScan，binary 名称为 motifscan。目标是实现一个适用于一代、二代、三代测序数据的高速 motif 检索工具。请直接在当前目录创建 Cargo.toml、src/main.rs、必要的模块文件、test/ demo 数据、benchmark/ 结果文件，并运行 cargo test 与 demo benchmark，确保能正常工作。

这个工具必须支持 FASTA、FASTQ、FASTA.GZ、FASTQ.GZ 输入，能够接受单个 motif，也能够接受 motif 表格文件。它要低内存、快速、多线程，支持 IUPAC 简并碱基、反向互补搜索、质量值过滤、TSV/CSV/TXT 输出，并提供 demo 数据和 benchmark 记录。

核心目标：
1. 输入可以是一代 Sanger 风格短 FASTA/FASTQ、Illumina 短 reads、Nanopore/PacBio 长 reads。
2. 输入 reads 可以很短，也可以几 kb、几十 kb、甚至更长。
3. motif 可以很短，例如 8–20 bp，也可以很长，例如 100–500 bp。
4. 程序不能把整个文件一次性读入内存，除非是未压缩文件且选择 mmap fast path。默认应使用 streaming parser。
5. 必须支持 --threads/-t 多线程。
6. 必须尽可能快，第一版至少实现 memchr-based exact matching fast path；IUPAC 模式可以走通用 bitmask matcher。
7. 输出要清楚、可复现、适合后续写进论文 methods。

命令行设计：

motifscan count \
  -i <input.fa/fq/fa.gz/fq.gz> \
  --motif <SEQUENCE> \
  --motif-name <NAME> \
  --motifs <motifs.tsv/csv/txt> \
  --revcomp \
  --iupac \
  --min-baseq <INT> \
  --min-mean-baseq <INT> \
  --threads <INT> \
  --output <FILE> \
  --output-format tsv|csv|txt \
  --report-read-hits <FILE>

motifscan classify \
  -i <input.fa/fq/fa.gz/fq.gz> \
  --motifs <motifs.tsv/csv/txt> \
  --revcomp \
  --iupac \
  --min-baseq <INT> \
  --min-mean-baseq <INT> \
  --threads <INT> \
  --output <FILE> \
  --summary <FILE> \
  --output-format tsv|csv|txt

参数要求：
- -i/--input 必须。
- --motif 和 --motifs 二选一。不能同时缺失。
- --motif-name 默认为 motif。
- --motifs 文件至少支持两列：name sequence。
- classify 模式下 --motifs 支持三列：name group sequence。
- --revcomp 表示同时搜索 motif 反向互补。
- --iupac 表示 motif 和 reads 中的 IUPAC 简并碱基按兼容规则匹配。
- --min-baseq 只对 FASTQ 有效，要求 motif window 内每个 base 的 Q 值都大于等于阈值。
- --min-mean-baseq 只对 FASTQ 有效，要求 motif window 平均 Q 值大于等于阈值。
- --threads/-t 默认使用 num_cpus。
- 输出默认 TSV。
- 对 FASTA 输入时，如果用户设置质量过滤参数，应给出 warning，但不要崩溃。
- 支持大小写不敏感匹配。

输出格式：

count 模式 summary 输出：
motif	sequence	length	reads_with_hit	total_hits	forward_hits	revcomp_hits

如果提供 --report-read-hits，则额外输出 read-level hits：
read_id	motif	strand	position	matched_sequence

classify 模式主输出：
group	motif	sequence	length	reads_with_hit	total_hits	forward_hits	revcomp_hits

classify summary 输出：
group	top_call	top_reads	second_call	second_reads	total_classified	top_ratio	ambiguous_reads	unclassified_reads

分类逻辑：
- 对每个 read，在同一个 group 内，如果只命中一个 motif，则将该 read 归类给该 motif。
- 如果同一个 group 内命中多个 motif，则计为 ambiguous。
- 如果同一个 group 内没有命中任何 motif，则计为 unclassified。
- total_hits 仍然记录所有 hit 数量。
- reads_with_hit 是至少命中该 motif 一次的 reads 数。
- 注意：一个 read 可以对不同 group 分别分类，例如同时有 18S 和 28S motif。
- 对 read-level 输出，记录所有 hits。

算法要求：
1. 实现 motif 预编译结构 CompiledMotif。
2. 对非 IUPAC、max mismatch = 0 的情况，使用 memchr 查找 motif 第一个碱基，然后局部比较 bytes。
3. 对 IUPAC 模式，将 motif 编译为 Vec<u8> bitmask，read base 也转 bitmask，用 motif_mask[i] & base_mask[j+i] != 0 判断。
4. 实现 reverse complement：
   A<->T, C<->G，同时支持 IUPAC：
   R<->Y, S<->S, W<->W, K<->M, B<->V, D<->H, N<->N。
5. reverse complement motif 不应重复统计 palindromic motif。若 motif 与其反向互补相同，revcomp_hits 仍然不要双倍计数。
6. 支持 overlapping hits，默认统计重叠命中。
7. FASTA/FASTQ parser 必须支持 multiline FASTA sequence。
8. FASTQ parser 至少支持标准 4 行 FASTQ。可以额外支持 multiline FASTQ，但若复杂度太高，先保证标准 FASTQ 稳定，并在 README 说明。
9. GZ 输入使用 flate2 streaming decoder。
10. 未压缩输入可以先用 BufReader streaming 实现；如果有余力，再加 memmap2 fast path，但不能牺牲稳定性。
11. 多线程实现方式：
    - 一个 reader 线程 streaming 读取 records。
    - 将 records 按 chunk 批量发送给 worker threads。
    - worker threads 扫描 motifs。
    - reducer 汇总结果。
    - 或者更简单：reader 读取 chunk Vec<Record>，使用 rayon par_iter 处理 chunk，然后汇总。
    - 必须避免每条 read 频繁锁全局 HashMap。每个线程/每个 chunk 做 local counts，然后 reduce。
12. 长 reads 情况要避免复制 sequence。Record 结构可拥有 Vec<u8>，处理时使用 &[u8]。
13. 尽量减少 String 转换。read_id 只有在 report-read-hits 时才需要写出完整 ID。
14. 对 motif 比 read 长的情况，直接跳过，不报错。
15. 对空 reads、空 motif、非法字符要给出合理错误或 warning。
16. 程序所有错误用 anyhow 返回，CLI 用 clap derive。
